-Goal-
Given a text document that is potentially relevant to this activity and a list of entity types, identify all entities of those types from the text and all relationships among the identified entities. 
<Important Supplement> Ensure every identified entity acts as either a source_entity or a target_entity in at least one relationship; do not retain "isolated entities" that have no connection to any other entity.
-Important Content Compliance Rules (MUST FOLLOW)-
1. Strictly base all outputs on the provided <Text> only; DO NOT add any information, assumptions, or extensions beyond the text content.
2. Focus on extracting factual, objective information from the text.
3. Entity descriptions and relationship explanations must be objective, neutral, and factual; avoid subjective evaluations, emotional language, or speculative statements.
4. If no valid entities/relationships are found, output an empty list followed by {completion_delimiter}—do not force-generate content.
5. <New Rule> After extracting entities and initial relationships, conduct a "entity-relationship check":
   a. List all extracted entities and verify whether each entity appears in at least one relationship (as source_entity or target_entity);
   b. For entities not in any relationship, re-examine the text to find potential connections with other entities (e.g., co-occurrence in the same sentence, shared attributes like "participated in the same project", "located in the same region");
   c. If text evidence supports a connection, add the corresponding relationship; if no text evidence exists for any connection, exclude the "isolated entity" from the final output (since it cannot contribute to the knowledge graph).

-Steps-
1. Identify all entities. For each identified entity, extract the following information:
- entity_name: Name of the entity, capitalized
- entity_type: One of the following types: [{entity_types}]
- entity_description: Comprehensive description of the entity's attributes and activities (based on the text)
Format each entity as ("entity"{tuple_delimiter}<entity_name>{tuple_delimiter}<entity_type>{tuple_delimiter}<entity_description>)
 
2. From the entities identified in step 1, identify all pairs of (source_entity, target_entity) that are *clearly related* to each other, and ensure:
   a. Every entity from step 1 is included in at least one relationship (as source_entity or target_entity);
   b. If an entity has no initial related pairs, recheck the text for hidden connections (e.g., "same event participation", "shared affiliation") and add valid relationships.
For each pair of related entities, extract the following information:
- source_entity: name of the source entity, as identified in step 1
- target_entity: name of the target entity, as identified in step 1
- relationship_description: explanation as to why you think the source entity and the target entity are related to each other (must be supported by the text)
- relationship_strength: a numeric score (1-10, 10 = strongest) indicating strength of the relationship between the source entity and target entity (e.g., "acquisition" = 8-10, "same project participation" = 2-4)

Format each relationship as ("relationship"{tuple_delimiter}<source_entity>{tuple_delimiter}<target_entity>{tuple_delimiter}<relationship_description>{tuple_delimiter}<relationship_strength>)
 
3. Return output in English as a single list of all the entities and relationships identified in steps 1 and 2 (excluding isolated entities with no text-supported relationships). Use **{record_delimiter}** as the list delimiter.
 
4. When finished, output {completion_delimiter}
